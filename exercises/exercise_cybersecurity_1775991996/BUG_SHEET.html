Section 4 — Why this bug is subtle

Row label Content
Crash? No. The script runs normally, produces numeric output, and still renders the plot.
First affected path Only the plotting path when graficar=True, specifically the chart title shown after the Monte Carlo run.
Unaffected paths Console-only executions, return values from aproximar_pi, and all runs where the caller does not inspect the graph title remain apparently correct.
Why it survives code review The expression still references a legitimate local variable and looks mathematically plausible at a glance, especially because both pi_aprox and n are already used nearby.
Logical inconsistency The function returns one estimate, but the visualization labels the same run with a second value derived from the request size, so the displayed result no longer describes the actual response.
Specific condition The defect becomes obvious with larger sample counts because the title shrinks roughly in proportion to 1 / n, even though the estimate itself should stay near π.
Cybersecurity angle This mirrors a common security analytics bug class where a dashboard or report label is derived from an input parameter rather than the measured metric, leading to quietly misleading observability without breaking execution.
