Debugging Exercise

Misreported plot metric during sampled analysis

📋 Statement

QA noticed something odd while reviewing a small Monte Carlo visualization that we sometimes use as a toy stand-in for sampled security telemetry. The command-line output looks reasonable, but when the plot is enabled, the number shown in the chart title is clearly inconsistent with the run that produced it. Instead of staying near the same estimate reported by the computation, the title becomes extremely small.

This only seems to happen on the graphing path. Plain runs that just print values to the console look normal, and the function still appears to return a plausible estimate. The issue is easiest to spot when increasing the sample size: the more points we analyze, the smaller the title value gets, even though the underlying estimate should not collapse like that.

From a security engineering perspective, this resembles a reporting integrity problem: a visualized metric is drifting with a request-sized parameter rather than reflecting the measured result of the sampled analysis. Please find and fix the backend logic so the displayed metric and the computed metric stay consistent.

🔴 Observed defective output

1) Baseline non-graph run

REQUEST
aproximar_pi(100, graficar=False)

RESPONSE
3.12
✅ Correct

The returned estimate is plausible and there is no visualization involved.

2) Same kind of run with graph enabled

REQUEST
aproximar_pi(100, graficar=True)

OBSERVED RESPONSE
return value: 3.12
plot title: "Aproximación de π ≈ 0.0312"
❌ Wrong

The function returns a plausible estimate, but the plot title reports a value smaller by a factor tied to the sample size.

3) Larger sampled run with graph enabled

REQUEST
aproximar_pi(1000, graficar=True)

OBSERVED RESPONSE
return value: 3.144
plot title: "Aproximación de π ≈ 0.003144"
❌ Wrong

The displayed value gets even smaller as the sample count grows, while the returned estimate stays near π.

4) Main script graph path

REQUEST
python main.py

OBSERVED RESPONSE
console output includes a plausible π estimate
final generated chart title for 5000 points shows a value near 0.0006 instead of a value near 3.14
❌ Wrong

The defect compounds with larger runs and is easiest to recognize in the final plotted example.

Samples sent Items processed Field value returned Field value shown in title Expected title value
100 100 3.12 0.0312 3.12
1000 1000 3.144 0.003144 3.144
5000 5000 3.1416 0.00062832 3.1416
